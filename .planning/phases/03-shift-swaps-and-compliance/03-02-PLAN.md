    1. Create src/lib/dal/swaps.ts with auth-gated queries:
       - `getOpenShifts()`: SELECT from shifts WHERE status = "open", JOIN with stores for context. Return array of { id, date, startTime, endTime, roleName, storeId }.
       - `getSwapRequests(filter?: { status?: string })`: SELECT from swapRequests, JOIN shifts (requestorShift, targetShift via aliases), JOIN users (requestor, targetEmployee, reviewer). Return array with full details: swap id, status, reason, createdAt, reviewedAt, requestor (id, name), requestorShift (id, date, startTime, endTime, roleName), targetEmployee (id, name), targetShift (id, date, startTime, endTime, roleName), reviewer name. For employees (role="employee"), filter to only show swaps where they are requestor or target.
       - `getPickupRequests(filter?: { status?: string })`: SELECT from shiftPickups JOIN shifts JOIN users. Return array with pickup id, status, createdAt, employee (id, name), shift details (date, startTime, endTime, roleName). For employees, filter to only their own requests.
       - `getEmployeeWeekShifts(employeeId: number, weekStartDate: string)`: Get all shifts for an employee in a given week (for hours calculation in swap validation). Use the same date range logic as getShiftsForWeek but filter by employeeId.

    2. Create src/lib/actions/swaps.ts:
       - `createSwapRequest(data: { requestorShiftId: number, targetShiftId: number })`:
         - Auth check: any authenticated user
         - Zod validate input
         - Verify requestor owns requestorShiftId (shift.employeeId === session.user.id)
         - Verify targetShift exists and is assigned to someone (that becomes targetEmployeeId)
         - Insert into swapRequests with status "pending"
         - Call logAuditEvent({ action: "swap.requested", ... })
         - revalidatePath("/swaps")
       - `approveSwap(swapId: number)`:
         - Auth check: manager or supervisor only
         - Fetch swap request, verify status is "pending" (optimistic locking)
         - Get week shifts for both employees using getEmployeeWeekShifts
         - Call validateSwapHours -- if invalid, auto-reject with the reason and return { success: false, message: reason }
         - If valid: update swapRequests set status="approved", reviewedBy, reviewedAt
         - Swap the employeeIds on the two shifts (update shift A's employeeId to targetEmployee, shift B's employeeId to requestor)
         - Call logAuditEvent({ action: "swap.approved", ... })
         - revalidatePath("/swaps") and revalidatePath("/schedule")
       - `rejectSwap(swapId: number, reason?: string)`:
         - Auth check: manager or supervisor
         - Update swapRequests set status="rejected", reviewedBy, reviewedAt, reason
         - Call logAuditEvent({ action: "swap.rejected", ... })
         - revalidatePath("/swaps")
       - `cancelSwap(swapId: number)`:
         - Auth check: any authenticated user
         - Verify the current user is the requestor
         - Verify status is "pending"
         - Delete the swap request (or set status to "rejected" with reason "Cancelled by requestor")
         - Call logAuditEvent({ action: "swap.cancelled", ... })
         - revalidatePath("/swaps")

    3. Create src/lib/actions/pickups.ts:
       - `createPickupRequest(data: { shiftId: number })`:
         - Auth check: any authenticated user
         - Verify the shift exists and status is "open"
         - Verify the employee has the required role: query employeeRoles WHERE userId = session.user.id AND roleName = shift.roleName. If not found, return { success: false, message: "You don't have the [roleName] qualification" }
         - Insert into shiftPickups with status "pending"
         - Call logAuditEvent({ action: "pickup.requested", ... })
         - revalidatePath("/swaps")
       - `approvePickup(pickupId: number)`:
         - Auth check: manager or supervisor
         - Fetch pickup request, verify status is "pending"
         - Get employee's week shifts, calculate total hours + new shift hours using calculateShiftHours
         - If would exceed 40h, auto-reject with reason
         - If valid: update shiftPickups status="approved", reviewedBy, reviewedAt
         - Update shift: set employeeId to the employee, status to "assigned"
         - Call logAuditEvent({ action: "pickup.approved", ... })
         - revalidatePath("/swaps") and revalidatePath("/schedule")
       - `rejectPickup(pickupId: number, reason?: string)`:
         - Auth check: manager or supervisor
         - Update shiftPickups status="rejected", reviewedBy, reviewedAt
         - Call logAuditEvent({ action: "pickup.rejected", ... })
         - revalidatePath("/swaps")
